# Internationalization

Wave 4 adds XMLUI's managed i18n foundation: locale resolution, translation bundles, ICU-style
message formatting, expression formatters, and document direction handling.

---

## Locale Resolution

The active locale is available as `App.locale`. XMLUI resolves it in this order:

1. `<App locale="...">`
2. `App.setLocale(locale)` user override
3. persisted locale from `appGlobals.localePersistKey`
4. browser `navigator.languages`, matched against loaded bundles and configured locales
5. `appGlobals.defaultLocale` (default `"en"`)

`App.localeSource` tells you which source won. `App.availableLocales` returns registered bundle
locales plus configured `appGlobals.availableLocales`.

```xml
<App locale="de-DE" direction="auto">
  ...
</App>
```

`direction="auto"` sets `document.documentElement.dir` based on the active locale. Use explicit
`ltr` or `rtl` when the app needs to override detection.

## Locale Bundles

Bundles are flat maps from translation key to message pattern. Register them declaratively:

```xml
<App
  locale="en"
  localeBundles="{{
    en: {
      'app.hello': 'Hello {name}',
      'app.inbox': '{count, plural, one {# message} other {# messages}}'
    },
    de: {
      'app.hello': 'Hallo {name}'
    }
  }}">
```

or at runtime:

```xml
<Button onClick="App.registerLocaleBundles('/locales/de-DE.json')">
  Load German
</Button>
```

Accepted shapes:

| Shape | Example |
|---|---|
| URL string | `"/locales/de-DE.json"` |
| Array | `["/locales/en.json", "/locales/de-DE.json"]` |
| Bundle object | `{ locale: "de-DE", messages: { "app.hello": "Hallo" } }` |
| Locale map | `{ "de-DE": { "app.hello": "Hallo" } }` |

Missing keys emit `kind:"i18n"` / `code:"missing-key"` and render the key as fallback. Invalid
message patterns emit `code:"icu-parse-error"`. `strictI18n` escalates these diagnostics.

## Translating Text

Use `App.translate()` or its alias `App.t()` in expressions:

```xml
<Text>{App.translate('app.hello', { name: loggedInUser.name })}</Text>
<Text>{App.t('app.inbox', { count: messages.length })}</Text>
```

Use `<I18n>` when the translation should render directly:

```xml
<I18n key="app.hello" name="{loggedInUser.name}" />
<I18n key="app.inbox" count="{messages.length}" />
```

Extra props become formatting variables. A leading `:` in a prop name is stripped before formatting,
which keeps the component compatible with expression-style variable props.

## Markup Slots

Translations can contain lightweight slot placeholders such as `<link/>`. `<I18n>` replaces them
with matching named XMLUI slots:

```xml
<I18n key="app.acceptTerms">
  <property name="link">
    <Link to="/terms">terms</Link>
  </property>
</I18n>
```

Bundle:

```json
{
  "app.acceptTerms": "Please accept the <link/>."
}
```

## Formatting Helpers

The `App` namespace exposes locale-aware `Intl.*` wrappers:

| Helper | Example |
|---|---|
| `App.formatNumber(value, options?)` | `App.formatNumber(total)` |
| `App.formatCurrency(value, currency, options?)` | `App.formatCurrency(price, "EUR")` |
| `App.formatList(values, options?)` | `App.formatList(["A", "B", "C"])` |
| `App.formatRelativeTime(value, unit, options?)` | `App.formatRelativeTime(-2, "day")` |
| `App.compare(a, b, options?)` | sort comparator |
| `App.pluralRules(value, options?)` | returns `one`, `other`, etc. |

## Built-In Framework Strings

The first extracted built-in strings are Form button defaults:

| Key | Default |
|---|---|
| `xmlui.form.cancel` | `Cancel` |
| `xmlui.form.save` | `Save` |
| `xmlui.form.saving` | `Saving...` |
| `xmlui.form.validating` | `Validating...` |

Custom Form label props still win. Translation is only used when the label prop is omitted.

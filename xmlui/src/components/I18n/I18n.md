# I18n

`I18n` renders a translated message from the active locale bundle. Use it to display
localised strings in markup wherever you need reactive locale switching or slot-based
inline markup (links, styled spans, etc.) inside a translated string.

## Simple usage

```xmlui
<I18n key="welcome.greeting" />
```

The component looks up `"welcome.greeting"` in the loaded locale bundles and renders the
translated text. If the key is missing, the key itself is rendered so the UI is never
broken by a typo.

## With variables

Pass variables as props using the `:name="{expression}"` binding syntax:

```xmlui
<I18n key="welcome.hello" :name="{user.firstName}" />
```

Bundle entry (English):

```json
{ "welcome.hello": "Hello, {name}!" }
```

## With inline markup slots

When the translated string contains markup placeholders, define named slots as children.
Each `<ComponentName/>` placeholder in the bundle string is replaced with the matching
slot.

```xmlui
<I18n key="terms.intro">
  <Link slot="termsLink" to="/terms">terms of service</Link>
  <Link slot="privacyLink" to="/privacy">privacy policy</Link>
</I18n>
```

Bundle entry:

```json
{ "terms.intro": "Please read our <termsLink/> and <privacyLink/>." }
```

## Props

| Prop  | Type     | Required | Description |
|-------|----------|----------|-------------|
| `key` | `string` | ✔        | Translation key to resolve from the active locale bundle. |

Any additional props (prefixed with `:` for binding) are passed as variables to the ICU
pattern substitution in the bundle string.

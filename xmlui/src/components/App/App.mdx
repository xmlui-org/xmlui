%-PROP-START layout

Here are a few samples demonstrating the usage of the `layout` property. All samples use this markup, except the value of `App`'s layout and a few marked code snippets:

```ueml
<App layout="(specific layout value)">
  <!-- AppHeader omitted for "vertical" and "vertical-sticky" -->
  <AppHeader>
    <prop name="logoTemplate">
      <Heading level="h3" value="Example App"/>
    </prop>
  </AppHeader>
  <NavPanel>
    <NavLink label="Home" to="/" icon="home"/>
    <NavLink label="Page 1" to="/page1"/>
    <NavLink label="Page 2" to="/page2"/>
  </NavPanel>
  <Pages defaultRoute="/">
    <Page url="/">
      <List datasource="https://api.spacexdata.com/v3/history">
        <prop name="itemTemplate">
          <Card title="{$item.title}" subTitle="{$item.details}"/>
        </prop>
      </List>
    </Page>
    <Page url="/page1">
      <Text value="Page 1" />
    </Page>
    <Page url="/page2">
      <Text value="Page 2" />
    </Page>
  </Pages>
  <Footer>Powered by XMLUI</Footer>
</App>
```

%-PROP-START scrollWholePage

This boolean property specifies whether the whole page should scroll (true) or just the content area (false).
The default value is `true`.

```ueml copy /scrollWholePage="false"/
<App scrollWholePage="false">
  <NavPanel>
    <NavLink label="Home" to="/" icon="home"/>
  </NavPanel>
  <Pages defaultRoute="/">
    <Page url="/">
```

%-PROP-START loggedInUser

Stores information about the currently logged in user.
Currently, there is no restriction on what the user data must look like.

```ueml copy /loggedInUser="{{ name: 'Joe', token: '1234' }}"/
<App loggedInUser="{{ name: 'Joe', token: '1234' }}">
  <NavPanel>
    <NavLink label="Home" to="/" icon="home"/>
  </NavPanel>
  <Pages defaultRoute="/">
    <Page url="/">
      <Text value="User name: {loggedInUser.name}" />
      <Text value="User token: {loggedInUser.token}" />
    </Page>
  </Pages>
</App>
```

%-EVENT-START ready

This event fires when the `App` component finishes rendering on the page.
Use it as `onReady` when inlining it on the component.

```ueml copy /onReady="isAppReady = true"/
<App onReady="isAppReady = true">
  <var name="isAppReady" value="{false}"/>
  <Text value="{isAppReady ? 'App is ready' : 'Sadly, App is not ready'}" />
</App>
```

%-STYLE-START

The `App` component has these theme variables:

- `max‑content‑width‑App`: This theme variable defines the maximum [width](./appearance/common-units/#size-values) of the main content. If the main content is broader, the engine adds margins to keep the expected maximum size.
- `shadow‑header‑App`: This variable sets the [shadow](./appearance/common-units/#shadow-values) to display around the app's header.
- `shadow‑Pages‑App`: This variable sets the style of [shadow](./appearance/common-units/#shadow-values) to display around the app's navigation panel (menu).
- `width-navPanel-App`: This theme variable sets the [width](./appearance/common-units/#size-values) of the navigation panel (menu) when the app is displayed with one of the vertical layouts.

> **Note**: There are other app-related components,
> such as [*`AppHeader`*](./AppHeader), [*`NavPanel`*](./NavPanel), and [`Footer`](./Footer),
> that can be styled to establish the app's visual appearance.

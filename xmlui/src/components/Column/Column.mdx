%-PROP-START pinTo

This property allows the column to be pinned to the `left` or right `edge` of the table.

```ueml copy {2,6} /pinTo="left"/ /pinTo="right"/
<Table data='{[...]}'>
  <Column bindTo="id" width="50" pinTo="left" />
  <Column bindTo="name" width="500" />
  <Column bindTo="quantity" width="300" />
  <Column bindTo="unit" width="300"/>
  <Column bindTo="category" width="100" pinTo="right"/>
</Table>
```
Scroll the table contents horizontally to see how the pinned columns are displayed.

%-PROP-START template

This prop provides customization to render any sort of components in the cells of the column.
Access the data's attributes via the `$item` in a binding expression.

```ueml copy {3-5}
<Table data="https://api.spacexdata.com/v3/rockets">
    <Column header="Image" size="140px">
      <Image height="100px" fit="cover" src="{$item.flickr_images[0]}"/>
    </Column>
    <Column header="Name" bindTo="rocket_name"/>
</Table>
```

%-PROP-START width

The following example sets the second column to an absolute size (size pixels), while the first and third columns have star sizes:

```ueml copy {3}
<Table data='{[...]}'>
  <Column bindTo="name" canResize="true" width="3*" />
  <Column bindTo="quantity" width="100" minWidth="50px" maxWidth="500" />
  <Column bindTo="unit" width="*" />
</Table>
```

Check what happens when you resize table columns:

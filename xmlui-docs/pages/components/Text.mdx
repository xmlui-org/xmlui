# Text

The `Text` component displays textual information in a number of optional styles and variants.

You can learn more about this component in the [Working with Text](/learning/using-components/text) article.

## Props

### `maxLines`

This property determines the maximum number of lines the component can wrap to. If there is no space to display all the contents, the component displays up to as many lines as specified in this property.

```ueml copy
<Text
  maxWidth="120"
  backgroundColor="cyan"
  value="A long text that will likely overflow "
  maxLines="2" />
```

### `ellipses`

This property indicates whether ellipses should be displayed when the text is cropped (`true`) or not (`false`).
By default, its value is set to `true`.

```ueml copy {4}
<VStack width="200" gap="0.5rem">
  <Text
    backgroundColor="cyan"
    maxLines="1"
    ellipses="false">
    Though this long text does is about to crop!
  </Text>
  <Text
    backgroundColor="cyan"
    maxLines="1">
    Though this long text does is about to crop!
  </Text>
</VStack>
```

### `preserveLinebreaks`

This property indicates if linebreaks should be preserved when displaying text.

```ueml copy {5}
<HStack gap="1rem">
  <Text
    width="200"
    backgroundColor="cyan"
    preserveLinebreaks="true"
    value="(preserve) This long text
    with several line breaks
            does not fit into a viewport with a 200-pixel width." />
  <Text
    width="200"
    backgroundColor="cyan"
    value="(do not preserve) This long text
    with several line breaks
            does not fit into a viewport with a 200-pixel width." />
 </HStack>
```

You can observe the effect of using `preserveLinebreaks`:

> **Note**: Remember to use the `value` property of `Text`. Linebreaks are converted to spaces when nesting the text into the `Text` component.

### `value`

The text to be displayed. This value can be also set via the `children` props.

```ueml copy
<Text value="An example text" />
<Text>An example text</Text>
```

### `variant`

Optional string value that provides named presets for text variants with a unique combination of font style, weight, size, color and other parameters.

| Value         | Description                                                                          |
| :------------ | :----------------------------------------------------------------------------------- |
| `abbr`        | Represents an abbreviation or acronym                                                |
| `caption`     | Represents the caption (or title) of a table                                         |
| `cite`        | Is used to mark up the title of a cited work                                         |
| `code`        | Represents a line of code                                                            |
| `codefence`   | Handles the display of code blocks if combined with a `code` variant                 |
| `deleted`     | Represents text that has been deleted                                                |
| `em`          | Marks text to stress emphasis                                                        |
| `inserted`    | Represents a range of text that has been added to a document                         |
| `keyboard`    | Represents a span of text denoting textual user input from a keyboard or voice input |
| `marked`      | Represents text which is marked or highlighted for reference or notation             |
| `mono`        | Text using a mono style font family                                                  |
| `paragraph`   | Represents a paragraph                                                               |
| `placeholder` | Text that is mostly used as the placeholder style in input controls                  |
| `sample`      | Represents sample (or quoted) output from a computer program                         |
| `small`       | Represents side-comments and small print                                             |
| `sub`         | Specifies inline text as subscript                                                   |
| `subtitle`    | Indicates that the text is the subtitle of some other content                        |
| `sup`         | Specifies inline text as superscript                                                 |
| `strong`      | Contents have strong importance                                                      |
| `title`       | Indicates that the text is the title of some other content                           |
| `var`         | Represents the name of a variable in a mathematical expression                       |

## Styling

The `Text` component uses these theme variables to customize its appearance:

- `align-vertical`
- `border-radius`
- `color`
- `color-bg`
- `color-border`
- `color-decoration`
- `font-family`
- `font-size`
- `font-weight`
- `font-style`
- `font-stretch`
- `letter-spacing`
- `line-decoration`
- `line-height`
- `offset-decoration`
- `padding-horizontal`
- `padding-vertical`
- `style-border`
- `style-decoration`
- `thickness-border`
- `thickness-decoration`
- `transform`

You must combine these property names with `Text` and the variant's name. For example, you can set a light green background for the `abbr` variant and add some horizontal padding with these theme variable settings:

```
"color-bg-Text-abbr": "lightgreen",
"padding-horizontal-Text-abbr": "8px"
```

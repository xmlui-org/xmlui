# CB Drive API Definition

This article describes the syntax and semantics of the CB Drive API.

## API Endpoints

### Getting the List of Available Drives

**Endpoint**: GET `/drives`

**Response**: [DriveArray DTO](#drivearray-dto)

**Status Codes**: 200

**Endpoint semantics**:

```js
// ...
```

### Getting the Attributes of a Drive

**Endpoint**: GET `/drive/:driveid`

**Path parameters**:

- `driveId`: The unique ID of drive

**Response**: [Drive DTO](#drive-dto)

**Status Codes**: 200

**Endpoint semantics**:

```js
// ...
```

### Creating a New Drive

**Endpoint**: POST `/drives`

**Path parameters**:

**Request**: [NewDrive DTO](#newdrive-dto)

**Response**: [Drive DTO](#drive-dto)

**Status Codes**: 200

**Endpoint semantics**:

```js
// ...
```

### Updating the Attributes of a Drive

**Endpoint**: PUT `/drive/:driveid`

**Path parameters**:

- `driveId`: The unique ID of drive to update

**Request**: [Drive DTO](#drive-dto)

**Status Codes**: 
- 204: Drive updated
- 404: Drive not found

**Endpoint semantics**:

```js
// ...
```

### Deleting a Drive

**Endpoint**: DELETE `/drive/:driveid`

**Path parameters**:

- `driveId`: The unique ID of drive to delete

**Status Codes**: 
- 204: Drive successfully deleted
- 404: Drive not found

**Endpoint semantics**:

```js
// ...
```

### Getting the Event Log of a Drive

**Endpoint**: GET `/drive/:driveid/logs`

**Path parameters**:

- `driveId`: The unique ID of drive

**Response**: application/text

**Status Codes**: 200

**Endpoint semantics**:

```js
// ...
```

### Executing a Drive Command

**Endpoint**: PUT `/drive/:driveid/command/:commandId`

**Path parameters**:

- `driveId`: The unique ID of drive to send the command to
- `commandId`: A command to send to the drive (e.g., `testConnection`, `start`, `stop`, `enable`. etc.)

**Request**: Optional command parameters (application/json)

**Response**: Command result status (application/json)

**Status Codes**: 200

**Endpoint semantics**:

```js
// ...
```

# Logo

The `Logo` component represents an app's logo resource. Usually, you place this component in an `AppHeader`.

You can learn more about the application layouts and using logos in the [App Component](/learning/using-components/app-component/) article.

## Using Logo

The component checks the application manifest for a logo resource (SVG file). If found, the engine displays it in the viewport of the Logo component.

This sample shows the manifest of a simple app with a logo definition:

```json copy {5}
{
  "name": "Tutorial",
  "version": "0.0.1",
  "resources": {
    "logo": "resources/xmlui-logo.svg",
    "favicon": "resources/favicon.ico"
  }
}
```

The following sample defines a new logo template that adds the "MyApp" title text before the logo. It uses the `Logo` component within the template definition:

```ueml copy {6}
<App layout="horizontal">
  <AppHeader>
    <prop name="logoTemplate">
       <Fragment>
         <Heading level="h2" value="MyApp"/>
         <Logo/>
       </Fragment>
    </prop>
  </AppHeader>
  <NavPanel>
    <NavLink label="Home" to="/" icon="home"/>
  </NavPanel>
  <Pages defaultRoute="/">
    <Page url="/">
      <CHStack>
        (Sample content)
      </CHStack>
    </Page>
  </Pages>
</App>
```

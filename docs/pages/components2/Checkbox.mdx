# Checkbox

The `Checkbox` component allows users to make binary choices, typically between checked or unchecked.
It consists of a small box that can be toggled on or off by clicking on it.

To bind data to a `Checkbox`, use the XMLUI [Forms infrastructure](
../learning/using-components/forms
).

## Context Variables

This component does not have any context variables.

## Properties

### `autoFocus`

Set this property to `true` to automatically get the focus when the component is displayed.

### `enabled`

This boolean property indicates whether the checkbox responds to user events (i.e. clicks);
it is `true` by default.

### `indeterminate`

Setting this prop signals that the component is in an _intedeterminate state_.

This prop is commonly used if there are several other checkboxes linked to one checkbox and some items in that group of checkboxes are in a mixed state: at least one item has a different value compared to the rest.

The following sample binds the state of two checkboxes to one and updates the state of the top checkbox accordingly. When the states of the bound checkboxes are different, the top checkbox is set to indeterminate:

```ueml copy {4}
<VStack margin="1rem" gap="0.8rem" var.indeterminate="{false}">
  <Checkbox
    label="Indeterminate Checkbox"
    indeterminate="{indeterminate}"
    initialValue="{cb1.value}"
    readOnly="true" />
  <ChangeListener
    listenTo="{ { v1: cb1.value, v2: cb2.value } }"
    onDidChange="indeterminate = cb1.value !== cb2.value" />
  Group of checkboxes:
  <HStack gap="0.5rem">
    <Checkbox label="Checkbox #1" id="cb1" initialValue="true" />
    <Checkbox label="Checkbox #2" id="cb2" initialValue="false" />
  </HStack>
</VStack>
```

Try this sample by clicking the bottom group of checkboxes.

### `initialValue`

This property sets whether the checkbox should be displayed in the checked (`true`) or unchecked (`false`) state when it is displayed initially.

### `labelId`

You can specify the identifier of a component acting as its label. When you click the label, 
the component behaves as you clicked it.

### `readOnly`

Set this property to `true` to disallow changing the component value.

### `required`

Set this property to `true` to indicate it must have a value before submitting the containing form.

### `validationStatus`

With this property, you can set the checkbox's validation status to "none", "error", "warning", 
or "valid".

## API

### `setValue`

You can use this method to set the checkbox's current value programmatically (`true`: checked, `false`: unchecked).

```ueml copy {9,12,14}
<VStack margin="1rem" gap="0.5rem" var.changes="">
  <Checkbox 
    id="checkbox" 
    readOnly="true" 
    label="This checkbox can be set only programmatically" 
    onDidChange="changes += '+'" />
  <HStack gap="0.5rem">
    <Button 
      label="Check" 
      onClick="checkbox.setValue(true)" />
    <Button 
      label="Uncheck" 
      onClick="checkbox.setValue(false)" />
  </HStack>
  <Text>The checkbox is {checkbox.value ? "checked" : "unchecked"}</Text>
  <Text value="Changes: {changes}" />
</VStack>
```

### `value`

You can query this read-only API property to query the checkbox's current value (`true`: checked, `false`: unchecked).

See an example in the `setValue` API method.

## Events

### `didChange`

This event is triggered when the `Checkbox` is toggled due to user interaction. A read-only checkbox never fires this event, and it won't fire if the checkbox's value is set programmatically.

```ueml copy
<VStack margin="0.75rem" gap="0.5rem" verticalAlignment="center" var.changes="">
  <Checkbox label="Changeable" onDidChange="changes += '+'" />
  <Checkbox label="Readonly" readOnly="true" onDidChange="changes += '-'" />
  <Text value="Changes: {changes}" />
</VStack>
```

### `gotFocus`

This event is triggered when the `Checkbox` receives focus.

Click the `Checkbox` in the example demo to change the label text. Note how clicking elsewhere resets the text to the original.

```ueml copy {4,5}
<VStack var.focused="{false}" margin="0.75rem" gap="0.5rem" verticalAlignment="center">
  <Checkbox
    value="true"
    onGotFocus="focused = true"
    onLostFocus="focused = false"
  />
  <Text value="{focused === true ? 'I am focused!' : 'I have lost the focus!'}" />
</VStack>
```

## Styling

The `Checkbox` component uses these theme variables to customize its appearance:

- `color-bg-Checkbox-default`
- `color-border-Checkbox-default`
- `radius-Checkbox-default`
- `color-bg-Checkbox--disabled`
- `color-bg-indicator-Checkbox`
- `color-bg-checked-Checkbox`
- `color-bg-checked-Checkbox-warning`
- `color-bg-checked-Checkbox-success`
- `color-bg-checked-Checkbox-error`
- `color-border-checked-Checkbox`
- `color-border-checked-Checkbox-warning`
- `color-border-checked-Checkbox-error`

> **Note**: some generic styling options are not available for this component.
> Namely using any `padding` features like the following:
>
> ```
> <Checkbox padding="1rem" />
> ```
>
> If you need to add space around the component, use `margins` instead of `paddings`.

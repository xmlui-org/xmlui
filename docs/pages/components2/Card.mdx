# Card

The `Card` component is a container for cohesive elements, often rendered visually as a card.

## Using Card

`Card` is a container; it does not have any explicit properties.
You can nest the card's content into the `<Card>` tag:

```ueml copy
<Stack padding="1rem">
  <Card maxWidth="300px">
    <HStack verticalAlignment="center" gap="0.5rem">
      <Icon name="info" />
      <Text value="Information" variant="strong" />
    </HStack>
    <Text value="This is an example text" />
  </Card>
</Stack>
```

There are also prestyled properties one can make use of, detailed in the [Properties section](#properties).
Prestyled elements always appear above other children.

```ueml copy

  <Card
    avatarUrl="https://i.pravatar.cc/100"
    title="Example Title"
    subTitle="Predefined subtitle"
    maxWidth="300px"
  >
    <HStack gap="1rem" verticalAlignment="center">
      <Icon name="info"/>
      This is a card
    </HStack>
  </Card>
</Stack>
```

## Context Variables

This component does not have any context variables.

## Properties

### `avatarUrl`

This property specifies the URL of the image to display in the avatar.

### `linkTo`

The `linkTo` wraps the title in a `Link` component that is clickable to navigate.

### `showAvatar`

Show the avatar (`true`) or not (`false`).
If no [`avatarUrl`](#avatarurl) is specified then the Avatar will show the first letters of the [`title`](#title).

Note that in the demo below if the `avatarUrl` is specified, `showAvatar` is automatically set to true but can still be hidden.

```ueml copy /showAvatar="true"/
<Stack padding="1rem" gap="1rem">
  <Card maxWidth="300px" avatarUrl="https://i.pravatar.cc/100" />
  <Card maxWidth="300px" showAvatar="true" title="Example Card" />
  <Card maxWidth="300px" showAvatar="true" />
</Stack>
```

## API

This component does not provide any API.

## Styling

The `Card` component uses these theme variables to customize its appearance:

- `border-Card`
- `color-bg-Card`
- `color-border-Card`
- `padding-Card`
- `padding-horizontal-Card`
- `padding-vertical-Card`
- `radius-Card`
- `shadow-Card`
- `style-border-Card`
- `thickness-border-Card`

The following theme definition changes the visual appearance of the card:

```json
{
  "id": "custom",
  "name": "Custom Theme",
  "themeVars": {
    "color-bg-Card": "$color-surface-200",
    "radius-Card": "4px",
    "color-border-Card": "$color-surface-600",
    "thickness-border-Card": "2px",
    "padding-horizontal-Card": "2rem"
  }
}
```

# Theme Variables

Theme variables follow a naming convention that allows decode what components and visual traits they affect:

```
<propertyertyName>-<ComponentId>-<trait1>-<trait2>--<state1>--<state2>--<stateN>
```

In this convention, the `<ComponentId>` part is optional.

- Those theme variables that do not use the `<ComponentId>` segment are _app-bound_ variables, for they define visuals that can be applied to the entire application, including multiple types of components.
- The variables with a `<ComponentId>` segment are _component-bound_ variables, which influence only the component's appearance defined by the corresponding ID.

The `<propertyertyName>` segment names a property (and optionally a part of a component) the particular theme variable affects. This segment should use lowercase letters and dashes. The property name can be a single word or multiple words separated with a dash (`-`). Here are a few examples:

- `color-text`: Text color 
- `color-bg`: Background color
- `color-bg-indicator`: The background color of the "indicator" part of a specific component. 
- `text-size`: The size of the text
- `line-height`: The height of a text line
- `padding-horizontal`: The value declaring the horizontal padding space

The `<trait1>` and `<trait2>` segments describe one or two trait values regarding a particular property (`<propertyertyName>`).

For example, the `Button` component has two traits that define its appearance. The `themeColor` trait defines the set of colors to use with the button; its available values are `primary`, `secondary`, and `attention`. The `variant` trait specifies the general appearance of the button (whether it has a border at all, etc.) with the values of `solid`, `outlined`, or `ghost`. The `themeColor` is the first trait, and `variant` is the second. So, we can use (among the others) these theme variables with button colors:

- `color-bg-Button-primary-solid`: Defines the background color of a button that is set up to use the `primary` theme color, displayed with the `solid` variant.
- `color-border-Button-attention-ghost`: Specifies the border color of a ghost button displayed with the `attention` theme color.

The `<state1>`, ..., `<stateN>` values define visual states that may modify the appearance of a particular UI element or component (regarding a specific trait). Here are a few examples:

- `hover`: The mouse is hovered over the component
- `active`: The particular component is being activated by the user (for example, a user presses down the mouse button while over a button element)
- `focus`: The particular element (e.g., a textbox) currently has the focus

When defining a theme variable, you can define multiple state values (in any order). Let's see a few examples:

- `color-bg-Button-primary-solid--hover`: The background color to use with a `solid` button with the `primary` theme color while the mouse hovers over it.
- `color-border-Button-attention-ghost--focus`: Specifies the border color of a focused ghost button displayed with the `attention` theme color.

## App-bound Traits [#app-bound-traits]

Theme variables in this category are applied to the entire application, not just to particular components.

### Colors

XMLUI defines these variables representing base colors:
- `color-surface`: The color for the surface (background) areas
- `color-primary`: The primary color of the app (buttons, badges, checkboxes, etc.)
- `color-secondary`: The secondary color of the app (buttons, badges, checkboxes, etc.)
- `color-warn`: The color to use for warnings
- `color-danger`: The color for signing dangerous situations (for example, the color of a Delete button)
- `color-success`: The color for signing success

These are base colors; when you specify them, XMLUI automatically creates several shade variants (using 50, 100, 200, 300, 400, 500, 600, 700, 800, 900, and 950 as prefixes) from the lightest to the darkest.

For example, when you set the the `color-primary` variable to `#008000`, XMLUI creates these shades:

<Playground
  previewOnly
  name="Custom primary color shades"
  components={[swatch, palette]}
  defaultTheme="custom"
  themes={[
    {
      name: "Custom Theme",
      id: "custom",
        themeVars: {
          "color-primary": "#008000",
        },
      },
    ]}
  app={`
      <Palette name="primary" />
  `}
/>

When selecting the primary color, choose one representing the middle shade (500). Otherwise, you may end up with an unuseful set of shades. For example, setting `surface-color` to `#001000` will create too many dark shades.

## Miscellaneous

### Screen Size

- `max-content-width`: The theme variable defines the maximum width of the content. A large screen with broad content makes reading text hard because of the long lines. This variable allows for maximizing the content's width and adding margins around it.
- `media-max-width-phone`: The maximum width of the screen (in pixels) considered as a mobile device (phone) screen.
- `media-max-width-landscape-phone`: The maximum width of the screen (in pixels) considered a landscape phone screen.
- `media-max-width-tablet`: The maximum width of the screen (in pixels) considered a tablet screen. Smaller screens represent a phone, broader a desktop.
- `media-max-width-desktop`: The maximum width of the screen (in pixels) considered a desktop screen.
- `media-max-width-large-desktop`: The maximum width of the screen (in pixels) considered a large desktop screen.

### Dropwdowns

XMLUI has several components with dropdowns. The following theme variables help style them:
- `color-border-dropdown-item`: The border color of the items within the dropdown.
- `color-bg-dropdown-item--hover`: The hovered color of the dropdown items' background.
- `color-bg-dropdown-item--active`: The active (selected) color of the dropdown items' background.

### Outline

Most components sign their focused state with an outline around them. These theme variables help style them:
- `color-outline--focus`: The color of the outline
- `thickness-outline--focus`: The thickness of the outline
- `style-outline--focus`: The <SmartLink href={ COMMON_VISUAL_PROPERTY_UNITS_BORDER_STYLE }>style</SmartLink> of the outline
- `offset-outline--focus`: The offset value of the outline

## Visual States [#visual-states]

XMLUI defines these visual states to use with styling component appearance:

|Name|Description|
|-|-|
|`‑‑active`|The component is active. For example, the left mouse button is pressed while the pointer is over the component.|
|`‑‑disabled`|The component is disabled; it does not accept user actions|
|`‑‑focus`|The component has the keyboard focus and processes keypress-related events|
|`‑‑hover`|Indicates the state when the mouse (pointer) hovers above the component's client area|

Components may define their custom visual states besides the ones displayed in this table.

# TextArea

`TextArea` is a component that provides a multiline text input area.

The `TextArea` is an input control component and is often used in forms.
See the [Using Forms](../learning/forms.mdx) guide for details.

To add new lines to the input field press `Shift` + `Enter`.

## Props

### `autoFocus`

If this boolean prop is set to `true`, the `TextArea` input will be focused when appearing on the UI.

```ueml copy
<TextArea autoFocus="true" />
```

### `autoSize`

If set to `true`, this boolean property enables the `TextArea` to resize automatically based on the number of lines inside it.

> **Note**: If either `autoSize`, `maxRows` or `minRows` is set, the `rows` prop has no effect.

Write multiple lines in the `TextArea` in the demo below to see how it resizes automatically.

### `enterSubmits`

This optional boolean property toggles whether pressing the `Enter` key on the keyboard prompts the parent `Form` component to submit.
The default value is is `true`.

Press `Enter` after writing something in the `TextArea` in the demo below.
See [Using Forms](../learning/forms.mdx) for details.

```ueml copy

    <TextArea

      enterSubmits="true"
      initialValue="Suzy Queue, 4455 Landing Lange, APT 4, Louisville, KY 40018-1234"
    />
</Form>
```

### `resize`

This optional property specifies in which dimensions can the `TextArea` be resized by the user.

| Value        | Description                   |
| :----------- | :---------------------------- |
|  undefined   | No resizing                   |
| `horizontal` | Can only resize horizontally  |
| `vertical`   | Can only resize vertically    |
| `both`       | Can resize in both dimensions |

Drag the small resize indicators at the bottom right on each of the controls in the demo.

    <TextArea resize="both" />
</VStack>

## API

### `value`

You can query this read-only API property to get the input component's current value.

See an example in the `setValue` API method.

### `setValue`

You can use this method to set the component's current value programmatically.

```ueml copy {3, 9, 12}
<VStack gap="0.5rem">
    <TextArea
      id="textarea"
      readOnly="true"
    />
    <HStack gap="0.5rem">
      <Button 
        label="Add Text" 
        onClick="textarea.setValue('This is an example')" />
      <Button 
        label="Remove Text"

```

### `gotFocus`

This event is triggered when the `TextArea` receives focus.
Clicking on the `TextArea` in the example demo changes the label text.
Note how clicking elsewhere resets the text to the original.

```ueml copy {3-4}

    initialValue="{focused === true ? 'I got focused!' : 'I lost focus...'}"
    onGotFocus="focused = true"
    onLostFocus="focused = false"
    var.focused="{false}"
/>
```

## Styling

The `TextArea` component uses these theme variables to customize its appearance:

- `radius-Textarea`
- `color-text-Textarea`
- `color-bg-Textarea--disabled`
- `thickness-border-Textarea`
- `style-border-Textarea`
- `color-border-Textarea--disabled`
- `color-bg-Textarea--disabled`
- `color-text-Textarea--disabled`
- `color-border-Textarea-default`
- `color-border-Textarea-error`
- `color-border-Textarea-warning`
- `color-border-Textarea-success`
- `color-placeholder-Textarea`

To style all input controls not just this component, use `Input` instead of the `TextArea` segment:

```
color-border-Input: "#0033FF"
```

`Input` affects the following controls:

- [`Checkbox`](./Checkbox.mdx)
- [`NumberBox`](./NumberBox.mdx)
- [`RadioGroup`](./RadioGroup.mdx)
- [`RadioGroupOption`](./RadioGroupOption.mdx)
- [`Switch`](./Switch.mdx)
- [`TextBox`](./TextBox.mdx)

See [custom themes](../learning/themes/overview.mdx) for styling details.

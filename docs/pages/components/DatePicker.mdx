# DatePicker

The `DatePicker` component allows users to select a date from a graphical calendar interface.

## Context Variables

This component does not have any context variables.

## Properties

### `autoFocus`

If this boolean prop is set to `true`, the `DatePicker` will be focused when it appears on the UI.
The default is `false`.

### `dateFormat`

Define custom date formats using this property.

Formats handle years (`y`), months (`m` or `M`), days of the month (`d`).
Providing multiple placeholder letters changes the display of the date.

The table below shows the available date formats:

| Format     | Example    |
| :--------- | :--------- |
| MM/dd/yyyy | 05/25/2024 |
| MM-dd-yyyy | 05-25-2024 |
| yyyy/MM/dd | 2024/05/25 |
| yyyy-MM-dd | 2024-05-25 |
| dd/MM/yyyy | 25/05/2024 |
| dd-MM-yyyy | 25-05-2024 |
| yyyyMMdd   | 20240525   |
| MMddyyyy   | 05252024   |

```ueml copy /dateFormat="dd-MM-yyyy"/
<DatePicker dateFormat="dd-MM-yyyy" initialValue="05/25/2024" />
```

/>

## API

### `setValue`

You can use this method to set the component's current value programmatically.

```ueml copy {3, 9, 12}
<VStack gap="0.5rem">
  <DatePicker
    id="picker"
    readOnly="true"
  />
  <HStack gap="0.5rem">
    <Button 
      label="Add Text" 
      onClick="picker.setValue('05/25/2024')" />
    <Button 
      label="Remove Text" 
      onClick="picker.setValue('')" />
  </HStack>
</VStack>
```

You can query this read-only API property to get the input component's current value.

See an example in the `setValue` API method.

### `gotFocus`

This event is triggered when the 
`DatePicker`
 receives focus.

Clicking on the `DatePicker` in the example demo changes the label text.
Note how clicking elsewhere resets the text to the original.

```ueml copy {4-5}
<VStack gap="0.5rem" var.isFocused="false">
  <Text value="{isFocused === true ? 'DatePicker focused' : 'DatePicker lost focus'}" />
  <DatePicker
    onGotFocus="isFocused = true"
    onLostFocus="isFocused = false"
  />
</VStack>
```

<Playground

This event is triggered when the `DatePicker` loses focus.

(See the example in the [`gotFocus`](#gotfocus) event section.)

## Styling

- `radius-DatePicker`
- `color-text-DatePicker`
- `color-bg-DatePicker--disabled`
- `thickness-border-DatePicker`
- `style-border-DatePicker`
- `color-border-DatePicker--disabled`
- `color-bg-DatePicker--disabled`
- `color-text-DatePicker--disabled`
- `color-text-value-DatePicker`
- `color-border-DatePicker-default`
- `color-border-DatePicker-error`
- `color-border-DatePicker-warning`
- `color-border-DatePicker-success`
- `color-placeholder-DatePicker`
- `color-bg-menu-DatePicker`
- `shadow-menu-DatePicker`
- `radius-menu-DatePicker`
- `color-bg-item-DatePicker--hover`
- `color-bg-item-DatePicker--active`

```

```

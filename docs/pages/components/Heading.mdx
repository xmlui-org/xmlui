# Heading

The `Heading` component displays titles and section headers.

> **Note**: `Heading` follows the basic rules of the HTML heading elements (`<h1>`, ..., `<h6>`).

For the shorthand versions see their reference page: [H1](./H1), [H2](./H2), [H3](./H3), [H4](./H4), [H5](./H5), [H6](./H6).

## Properties

### `level`

This property sets the visual significance (level) of the heading.

| Value | Description                                           |
| :---- | :---------------------------------------------------- |
| `h1`  | **(default)** Equivalent to the `<h1 />` HTML element |
| `h2`  | Equivalent to the `<h2 />` HTML element               |
| `h3`  | Equivalent to the `<h3 />` HTML element               |
| `h4`  | Equivalent to the `<h4 />` HTML element               |
| `h5`  | Equivalent to the `<h5 />` HTML element               |
| `h6`  | Equivalent to the `<h6 />` HTML element               |

### `maxLines`

This property determines the maximum number of lines the component can wrap to. If there is no space to display all the contents, the component displays up to as many lines as specified in this property.

```ueml copy
<H2 
  maxWidth="160" 
  backgroundColor="cyan"
  value="A long heading text that will likely overflow" maxLines="2" />
```

### `ellipses`

This property indicates whether ellipses should be displayed (`true`) when the heading text is cropped or not (`false`).
The default value is `true`.

```ueml copy {4}
<VStack width="200" gap="0.5rem">
  <H3 
    backgroundColor="cyan" 
    maxLines="1"
    ellipses="false">
    Though this long text does is about to crop!
  </H3>
  <H3 
    backgroundColor="cyan" 
    maxLines="1">
    Though this long text does is about to crop!
  </H3>
</VStack>
```

### `preserveLinebreaks`

This property indicates if linebreaks should be preserved when displaying text.

```ueml copy {5}
<HStack gap="1rem">
  <H3
    width="200"
    backgroundColor="cyan"
    preserveLinebreaks="true"
    value="(preserve) This long text
    with several line breaks
            does not fit into a viewport with a 200-pixel width." />
  <H3
    width="200"
    backgroundColor="cyan"
    value="(do not preserve) This long text
    with several line breaks
            does not fit into a viewport with a 200-pixel width." />
 </HStack>
```
You can observe the effect of using `preserveLinebreaks`:

> **Note**: Remember to use the `value` property of `Heading`. Linebreaks are converted to spaces when nesting the text into the `Heading` component.

### `value`

This property value determines the text displayed in the heading. `Heading` accepts nest child components (text) instead of `value`. If both `value` and nested components are used, `value` is used.

```ueml copy
<VStack>
  <Heading value="This is level 3 (value)" level="h3" />
  <Heading level="h3">This is level 3 (child)</Heading>
  <Heading value="Value" level="h3"><Icon name="trash" /></Heading>
</VStack>
```

## Events

This component does not provide event handlers to any system or user event.

## Styling

The `Heading` component uses these theme variables to customize its appearance:

- `color-decoration`
- `color-text`
- `font-family`
- `font-weight`
- `font-size`
- `letter-spacing`
- `line-decoration`
- `line-height`
- `margin-top`
- `margin-bottom`
- `offset-decoration`
- `style-decoration`
- `thickness-decoration`

Most theme variables above can be used with `Heading` and `H1`...`H6` component specifiers, except the following, which are bound to `H1`...`H6` only (they have no effect with `Heading`):

- `font-size`
- `line-height`
- `margin-top`
- `margin-bottom`

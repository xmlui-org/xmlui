# Select

A `Select` component defines a dropdown list with label-value pairs. The dropdown list displays the labels. XMLUI stores the selected value for further use when the user selects a particular item.

## Using `Select`

The component accepts `Option` elements as children defining a particular option's label-value pair. `Option` requires a `value` property and optionally has a `label` to display. If `label` is not specified, `value` is shown.

```ueml copy
<Select>
  <Option value="opt1" label="first"/>
  <Option value="opt2" label="second"/>
  <Option value="opt3" label="third"/>
</Select>
```

### `emptyListTemplate`

This optional property provides the ability to customize what is displayed when the list of options is empty.

Click on the second field to see the custom empty list indicator.

```ueml copy {9-11}

    <Select>
```

### `initialValue`

The initial value displayed in the input field.
Note that the `initialValue` prop does not display anything in the field if it is not found among the available `options`.

```ueml copy
<Select initialValue="opt3">
  <Option value="opt1" label="first"/>
  <Option value="opt2" label="second"/>
  <Option value="opt3" label="third"/>
</Select>
```

### `optionTemplate`

This property enables the customization of list items.
To access the attributes of a list item use the `$item` keyword.

```ueml copy
<Select>
  <prop name="optionTemplate">
    <HStack verticalAlignment="center" gap="0.5rem">
      <Icon name="info" />
      <Text value="{$item.label}" variant="strong" />
    </HStack>
  </prop>
  <Option value="opt1" label="first"/>
  <Option value="opt2" label="second"/>
  <Option value="opt3" label="third"/>
</Select>
```

### `validationStatus`

This prop is used to visually indicate status changes reacting to form field validation.

| Value     | Description                                           |
| :-------- | :---------------------------------------------------- |
| `valid`   | Visual indicator for an input that is accepted        |
| `warning` | Visual indicator for an input that produced a warning |
| `error`   | Visual indicator for an input that produced an error  |

```ueml copy
<VStack gap="0.5rem">

  <Select validationStatus="warning" />
  <Select validationStatus="error" />
```

## API

### `value`

You can query this read-only API property to get the input component's current value.

See an example in the `setValue` API method.

### `setValue`

You can use this method to set the component's current value programmatically.

```ueml copy {2, 7-8}
<VStack gap="1rem">
  <Select id="select">
    <Option label="First Item" value="first"/>
    <Option label="Second Item" value="second"/>
  </Select>
  <HStack gap="0.5rem">
    <Button label="Set First" onClick="select.setValue('first')" />
    <Button label="Set Second" onClick="select.setValue('second')" />
  </HStack>
</VStack>
```

## Events

### `didChange`

This event is triggered after the user has changed the field value.

Write in the input field and see how the `Text` underneath it is updated in parallel.

```ueml copy /onDidChange="(val) => field = val"/
<VStack gap="0.5rem" var.field="">
  <Select initialValue="{field}" onDidChange="(val) => field = val">
    <Option value="opt1" label="first"/>
    <Option value="opt2" label="second"/>
    <Option value="opt3" label="third"/>
  </Select>
  <Text value="{field}" />
</VStack>
```

### `gotFocus`

This event is triggered when the `Select` receives focus.
Clicking on the `Select` in the example demo changes the label text.
Note how clicking elsewhere resets the text to the original.

```ueml copy {2-3}
<Select
  onGotFocus="focused = true"
  onLostFocus="focused = false"
  placeholder="{focused === true ? 'I got focused!' : 'I lost focus...'}"
  var.focused="{false}"
/>
```

## Styling

The `Select` component uses these theme variables to customize its appearance:

- `radius-Select`
- `color-text-Select`
- `color-bg-Select--disabled`
- `thickness-border-Select`
- `style-border-Select`
- `color-border-Select--disabled`
- `color-bg-Select--disabled`
- `color-text-Select--disabled`
- `color-border-Select-default`
- `color-border-Select-error`
- `color-border-Select-warning`
- `color-border-Select-success`
- `color-placeholder-Select`

```json
"radius-Select": "30px",
"color-placeholder-Select": "red",
"thickness-border-Select": "4px"
```

<Playground
  name="Example: styling"

To style all input controls not just this component, use `Input` instead of the `Select` segment:

```
color-border-Input: "#0033FF"
```

`Input` affects the following controls:

- [`Checkbox`](./Checkbox.mdx)
- [`NumberBox`](./NumberBox.mdx)
- [`RadioGroupOption`](./RadioGroupOption.mdx)
- [`Switch`](./Switch.mdx)
- [`TextArea`](./TextArea.mdx)
- [`TextBox`](./TextBox.mdx)

See [custom themes](../learning/themes/overview.mdx) for styling details.

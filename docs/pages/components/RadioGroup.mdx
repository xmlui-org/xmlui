# RadioGroup

The `RadioGroup` input component is a group of radio buttons ([`RadioGroupOption`](
./RadioGroupOption.mdx
)
components) that allow users to select only one option from the group at a time.

`RadioGroup` is often used in forms. See the [Using Forms](/learning/using-components/forms.mdx) guide
for details.

## Context Variables

This component does not have any context variables.

## Properties

### `autoFocus`

Set this property to `true` to automatically get the focus when the component is displayed.

### `enabled`

This property indicates whether the input accepts user actions (`true`) or not (`false`). The default value is `true`.

```ueml copy /enabled="false"/
<RadioGroup initialValue="first" enabled="false">
  <HStack gap="1rem" padding="1rem">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
    <RadioGroupOption label="Third Item" value="third"/>
  </HStack>
</RadioGroup>
```

### `labelId`

You can specify the identifier of a component acting as its label. When you click the label,
the component behaves as you clicked it.

### `maxLength`

This property sets the maximum number of characters you can type into the component's text.

### `readOnly`

Set this property to `true` to disallow changing the component value.

### `required`

Set this property to `true` to indicate it must have a value before submitting the containing form.

### `validationStatus`

With this property, you can set the checkbox's validation status to "none", "error", "warning",
or "valid".

This prop is used to visually indicate status changes reacting to form field validation.

| Value     | Description                                           |
| :-------- | :---------------------------------------------------- |
| `valid`   | Visual indicator for an input that is accepted        |
| `warning` | Visual indicator for an input that produced a warning |
| `error`   | Visual indicator for an input that produced an error  |

```ueml copy
<HStack gap="1rem" padding="1rem">
  <RadioGroup initialValue="first" validationStatus="error">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
  </RadioGroup>
  <RadioGroup initialValue="first" validationStatus="warning">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
  </RadioGroup>
  <RadioGroup initialValue="first" validationStatus="valid">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
  </RadioGroup>
</HStack>
```

## API

### `setValue`

You can use this method to set the component's current value programmatically.

```ueml copy {2, 7-8}
<VStack gap="1rem">
  <RadioGroup id="radio">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
  </RadioGroup>
  <HStack gap="0.5rem">
    <Button label="Set First" onClick="radio.setValue('first')" />
    <Button label="Set Second" onClick="radio.setValue('second')" />
  </HStack>
</VStack>
```

### `value`

You can query this read-only API property to get the input component's current value.

See an example in the 
`setValue`
 API method.

## Events

### `didChange`

This event is triggered after the user has changed the field value. The following example uses this event to display the selected option's value:

```ueml copy /onDidChange="(val) => field = val"/
<VStack var.field="" gap="0.5rem" padding="1rem">
  <RadioGroup initialValue="{field}" onDidChange="(val) => field = val">
    <RadioGroupOption label="First Item" value="first"/>
    <RadioGroupOption label="Second Item" value="second"/>
  </RadioGroup>
  <Text value="{field}" />
</VStack>
```

### `gotFocus`

This event is triggered when the 
`RadioGroup`
 receives focus.

### `lostFocus`

This event is triggered when the 
`RadioGroup`
 loses focus.

## Styling

`RadioGroup` is a component that governs its children and stores the selected value. It does not support styling; however, you can style the options within a group.

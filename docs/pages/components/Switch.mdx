# Switch

The 
`Switch`
 component is a user interface element that allows users to toggle between two states: on and off.
It consists of a small rectangular or circular button that can be moved left or right to change its state.
It's analogous in function to the [
`Checkbox`
](
./Checkbox.mdx
).

To bind data to a 
`Switch`
, use the XMLUI [Forms infrastructure](
../learning/using-components/forms
).

## Context Variables

This component does not have any context variables.

## Properties

### `autoFocus`

Set this property to `true` to automatically get the focus when the component is displayed.

### `enabled`

it is `true` by default.

### `initialValue`

This property sets whether the switch should be displayed in the checked (`true`) or unchecked 
(`false`) state when it is displayed initially.

### `label`

This property sets the label of the component.

```ueml copy
  <Switch label="Another label" intialValue="false" />
```

You can specify the identifier of a component acting as its label. When you click the label, 
the component behaves as you clicked it.

### `readOnly`

Set this property to `true` to disallow changing the component value.

If true, the value of the component cannot be modified.

```ueml copy

  <Switch readOnly="true" label="Unchecked" intialValue="false" />
```

### `required`

Set this property to `true` to indicate it must have a value before submitting the containing form.

### `validationStatus`

With this property, you can set the checkbox's validation status to "none", "error", "warning", 
or "valid".

## API

### `setValue`

You can use this method to set the switch's current value programmatically (`true`: checked, `false`: unchecked).

```ueml copy {9,12,14}
<VStack margin="1rem" gap="0.5rem" var.changes="">
  <Switch 
    id="mySwitch" 
    readOnly="true" 
    label="This switch can be set only programmatically" 
    onDidChange="changes += '+'" />
  <HStack gap="0.5rem">
    <Button 
      label="Check" 
      onClick="mySwitch.setValue(true)" />
    <Button 
      label="Uncheck" 
      onClick="mySwitch.setValue(false)" />
  </HStack>
  <Text>The switch is {checkbox.value ? "checked" : "unchecked"}</Text>
  <Text value="Changes: {changes}" />
</VStack>
```

### `value`

You can query this read-only API property to query the switch's current value (`true`: checked,
`false`: unchecked).

See an example in the 
`setValue`
 API method.

## Events

### `didChange`

This event is triggered when the `Switch` is toggled due to user interaction.
A read-only switch never fires this event, and it won't fire if the switch's value is set programmatically.

```ueml copy
<VStack margin="0.75rem" gap="0.5rem" verticalAlignment="center" var.changes="">
  <Switch label="Changeable" onDidChange="changes += '+'" />
  <Switch label="Readonly" readOnly="true" onDidChange="changes += '-'" />
  <Text value="Changes: {changes}" />
</VStack>
```

### `gotFocus`

This event is triggered when the `Switch` receives focus.

Click the `Switch` in the example demo to change the label text. Note how clicking elsewhere resets the text to the original.

```ueml copy {4,5}
<HStack var.focused="{false}" margin="0.75rem" gap="0.5rem" verticalAlignment="center">
    <Switch
      value="{true}"
      onGotFocus="focused = true"
      onLostFocus="focused = false"
    />
    <Text value="{focused === true ? 'I got focused!' : 'I lost focus...'}" />
</HStack>
```

      />
      <Text value="{focused === true ? 'I am focused!' : 'I have lost the focus!'}" />

### `lostFocus`

This event is triggered when the 
`Switch`
 loses focus.

(See the example above)

## Styling

The `Switch` component uses these theme variables to customize its appearance:

- `color-bg-Switch`
- `color-border-Switch`
- `color-bg-Switch--disabled`
- `color-bg-indicator-Switch`
- `color-bg-checked-Switch`
- `color-bg-checked-Switch-warning`
- `color-bg-checked-Switch-success`
- `color-bg-checked-Switch-error`
- `color-border-checked-Switch`
- `color-border-checked-Switch-warning`
- `color-border-checked-Switch-success`
- `color-border-checked-Switch-error`

> **Note**: some generic styling options are not available for this component.
> Namely using any `padding` features like the following:
>
> ```
> <Switch padding="1rem" />
> ```
>
> If you need to add space around the component, use `margins` instead of `paddings`.

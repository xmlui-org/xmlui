# Switch

The `Switch` component is a user interface element that allows users to toggle between two states: on and off. It consists of a small rectangular or circular button that can be moved left or right to change its state.

## Properties

### `autoFocus`

This property sets the label of the component.

Default value: `false`.

### `description`

(*** NOT IMPLEMENTED YET ***) This optional property displays an alternate description of the Switch besides its label.

### `enabled`

This boolean property value indicates whether the component responds to user events (`true`) or not (`false`).

Default value: `true`.

This boolean property indicates whether the checkbox responds to user events (i.e. clicks);
it is `true` by default.

```ueml copy {4-5, 9-10}
<VStack margin="1rem" gap="0.8rem">
  Enabled switches:
  <HStack gap="0.5rem">
    <Switch initialValue="true" enabled="true" />
    <Switch initialValue="false" enabled="true" />
  </HStack>
  Disabled switches:
  <HStack gap="0.5rem">
    <Switch initialValue="true" enabled="false" />
    <Switch initilaValue="false" enabled="false" />
  </HStack>
</VStack>
```

### `indeterminate`

The `true` value of this property signals that the component is in an _intedeterminate state_.

### `initialValue`

This property sets the component's initial value.

Default value: `false`.

### `label`

This property sets the label of the component.

This property sets the label of the component.

### `labelId`

You can specify the identifier of a component acting as its label. When you click the label, the component behaves as you clicked it.

Places the label at the given position of the component.

Default value: `"right"`.

Available values: `top`, `right` **(default)**, `bottom`, `left`

### `validationStatus`

This property allows you to set the validation status of the input component.

Available values:

| Value | Description |
| --- | --- |
| `valid` | Visual indicator for an input that is accepted |
| `warning` | Visual indicator for an input that produced a warning |
| `error` | Visual indicator for an input that produced an error |


## Events

### `click`

This event is triggered when the Switch is clicked.

### `didChange`

This event is triggered when value of Switch has changed.

This event is triggered when the `Switch` is toggled due to user interaction.
A read-only switch never fires this event, and it won't fire if the switch's value is set programmatically.

```ueml copy
<VStack margin="0.75rem" gap="0.5rem" verticalAlignment="center" var.changes="">
  <Switch label="Changeable" onDidChange="changes += '+'" />
  <Switch label="Readonly" readOnly="true" onDidChange="changes += '-'" />
  <Text value="Changes: {changes}" />
</VStack>
```

### `gotFocus`

This event is triggered when the Switch has received the focus.

This event is triggered when the `Switch` receives focus.

Click the `Switch` in the example demo to change the label text. Note how clicking elsewhere resets the text to the original.

```ueml copy {4,5}
<HStack var.focused="{false}" margin="0.75rem" gap="0.5rem" verticalAlignment="center">
    <Switch
      value="{true}"
      onGotFocus="focused = true"
      onLostFocus="focused = false"
    />
    <Text value="{focused === true ? 'I got focused!' : 'I lost focus...'}" />
</HStack>
```

<Playground

        onLostFocus="focused = false"
      />
      <Text value="{focused === true ? 'I am focused!' : 'I have lost the focus!'}" />
    </VStack>
  `}
/>

### `lostFocus`

This event is triggered when the Switch has lost the focus.

## Context Values

This component does not have any context values.

## Exposed Members

### `setValue`

You can use this method to set the component's current value programmatically (`true`: checked, `false`: unchecked).

### `value`

You can query this read-only API property to query the component's current value (`true`: checked, `false`: unchecked).

## Styling

The `Switch` component uses these theme variables to customize its appearance:

- `color-bg-Switch`
- `color-border-Switch`
- `color-bg-Switch--disabled`
- `color-bg-indicator-Switch`
- `color-bg-checked-Switch`
- `color-bg-checked-Switch-warning`
- `color-bg-checked-Switch-success`
- `color-bg-checked-Switch-error`
- `color-border-checked-Switch`
- `color-border-checked-Switch-warning`
- `color-border-checked-Switch-success`
- `color-border-checked-Switch-error`

> **Note**: some generic styling options are not available for this component.
> Namely using any `padding` features like the following:
>
> ```
> <Switch padding="1rem" />
> ```
>
> If you need to add space around the component, use `margins` instead of `paddings`.

# Layout Properties

Each component has a particular visual appearance by default; this default display is often the best the component can have in a particular context. However, as we change the layout of an application, we may need to change some visual properties of components. Layout properties allow you to set up the visual appearance of components on a component instance base.

This code snippet renders a text:

## Dimensions

_We should change the Text and Heading (and other text-like) components to have inline rendering by default. When it's done, this section can be completed._

When the engine renders particular components, it determines their dimensions according to the type of a particular component and its explicitly set dimension properties.

### Inline and Block Rendering

XMLUI has components dedicated to establishing the layout of their child elements, called layout containers. Unless a layout container is used, the engine renders components from left to right and top to bottom.

> **Note**: You can use the `direction` layout property to support languages that use right-to-left reading. If this is set to `rtl` (right-to-left), the engine renders components from right to left and top to bottom.

The engine puts components side-by-side while they fit into the available viewport width. The engine starts a new line if a component does not fit into the current line.

The engine renders any components either inline or in a block. Inline rendering means the engine does not start on a new line and only takes up as much width as necessary. On the other hand, block rendering always starts on a new line and takes up the entire viewport width available.

The type of a component determines how the engine renders it. Some components, like `Heading` and `Stack`, suggest block rendering, while others, like `Text` and `Button`, are rendered inline.

```ueml copy
<Fragment>
  <H3 backgroundColor="cyan">This is a heading</H3>
  <Text backgroundColor="magenta">Before button</Text>
  <Button label="Click me!" backgroundColor="green"/>
  <Text backgroundColor="magenta">After button</Text>
</Fragment>
```

### Specifying Dimensions

You can explicitly specify several dimensions of a particular component with the `width`, `height`, `minWidth`, `minHeight`, `maxWidth`, and `maxHeight` layout properties.

The dimension properties use <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size values</SmartLink>, which contain a value and a unit that together determine the actual size. Here are a few examples of size values:

```text
12px
0.25rem
50%
0
125mm
```

By default (when the unit is omitted), XMLUI assumes pixels (`px`).

The XMLUI percentage value is relative to the size of the viewport a parent container offers for a particular child element. Here is an example to demonstrate how percentage values are used:

## Overflow

A component may not fit into the area its parent provides. This is called overflow.
XMLUI provides a few properties to define how a particular component should behave in situations like these.
Individual components, such as <SmartLink href={COMPONENT_TEXT}>`Text`</SmartLink>, <SmartLink href={COMPONENT_HEADING}>`Heading`</SmartLink>, and others, provide further techniques to handle overflow.

|Property|Description|
|-|-|
|`horizontalOverflow`| This property defines the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_OVERFLOW}>overflow</SmartLink> strategy when a component horizontally overflows the UI its parent provides. |
|`verticalOverflow`| This property defines the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_OVERFLOW}>overflow</SmartLink> strategy when a component vertically overflows the UI its parent provides. |

### Horizontal Overflow

The following example demonstrates how the horizontal overflow property works:

```ueml copy

    <Text maxLines="1">
      This is a long text that probably won't fit entirely in a single line.
    </Text>
  </HStack>
  <HStack width="200px" height="40px" backgroundColor="lightgreen" horizontalOverflow="scroll">
    <Text color="purple" width="1000px">
      This is a long text that probably won't fit entirely in a single line.
    </Text>
  </HStack>
  <HStack width="200px" height="40px" backgroundColor="silver">
    <Text color="purple">
      This is a long text that probably won't fit entirely in a single line.
    </Text>
  </HStack> 
</HStack>
```

Observe the second text's width, which is set to 1000px, to ensure it won't be trimmed and closed with ellipses.
Due to the `horizontalOverflow="scroll"` setting, you can see a horizontal scrollbar at the bottom of the light green box:

## The Box Model

Most components use a rectangular space (or box) their parent container provides.
Several layout properties define how the component should use this rectangular space.
These properties follow the so-called *box model* which comes from the CSS model of the same name.

This model works with the following boxes nested in each other (see an example for each underneath the list):
- **Component box**: The area the parent provides for the component (the size of this box depends on the rendering logic of the parent).
- **Margin box**: Within the component box, the component may define a margin that surrounds the other boxes of the component. The margin box displays the background (background color) the parent component provides.
- **Border box**: Within the margin box, the component may have a border with a particular width. Depending on the border pattern (for example, in the case of a dashed or dotted), the component background may be visible under the border.
- **Padding box**: The component may define a padding surrounding its content. The background of the padding box is set to the component background.
- **Content box**: This is the rectangular area where the component will render its child components.

The engine defines several layout properties to set up the box model of a particular component. Here is an example:

```ueml copy
<VStack height="100%" backgroundColor="cyan">
  <VStack height="85%" 
    backgroundColor="red"
    margin="20" 
    padding="60"
    border="8px dashed blue">
    <VStack width="100%" height="100%" backgroundColor="green" />
  </VStack>  
</VStack>
```

The child with the red background has a margin of `20px` and is smaller than its parent (its height is also 85% of its parent).
This child also has a dashed border (`border="8 dashed blue"`),
and has a padding of `60px` that squishes its own child to occupy less space inside of it (the one with the green background).
Finally, the `VStack` with the green background fills in the available content area defined by its parent with the red background.

### Margin

You can use these properties to define component margins:

| Property | Description |
|-|-|
| `margin` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the margin area; uses the same value for the left, top, right, and bottom margins. |
| `horizontalMargin` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the horizontal margins (left and right) to the specified value. |
| `verticalMargin` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the vertical margins (top and bottom) to the specified value. |
| `marginLeft` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the left margin to the specified value. |
| `marginTop` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the top margin to the specified value. |
| `marginRight` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the right margin to the specified value. |
| `marginBottom` | Sets the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_SIZE}>size</SmartLink> of the bottom margin to the specified value. |

When you decorate a component with multiple properties setting the same margin, the most specific one prevails. For example, `marginLeft` overrides `horizontalMargin` and `margin`.

### Border

You can use these properties to define component borders:

| Property | Description |
|-|-|
| `border` | Sets the value of the <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_BORDER}>border</SmartLink> area; uses the same value for the left, top, right, and bottom borders. |

| `borderTop` | Sets the value of the top <SmartLink href={COMMON_VISUAL_PROPERTY_UNITS_BORDER}>border</SmartLink> to the specified one. |

| Property | Description |

If you set `radius` and any other properties, the others overwrite `radius` for the particular corner.

# Internationalization (i18n) — AI Reference

Wave 4 adds the managed i18n surface for XMLUI: locale resolution, translation bundles, ICU-style
message formatting, expression formatters, and document direction handling.

## Core Files

| File | Role |
|---|---|
| `xmlui/src/components-core/i18n/locale-resolver.ts` | BCP-47 validation, normalization, active-locale priority |
| `xmlui/src/components-core/i18n/bundle-store.ts` | Locale bundle registration and lookup with language fallback |
| `xmlui/src/components-core/i18n/icu.ts` | Local ICU-style formatter for variables, plural/select, number/date/time |

## Locale Resolution

Active locale priority:

1. `<App locale="...">`
2. `App.setLocale(locale)` user override
3. persisted locale from `appGlobals.localePersistKey`
4. `navigator.languages` first match against loaded bundles / `appGlobals.availableLocales`
5. `appGlobals.defaultLocale` (default `"en"`)

`App.locale` and `App.localeSource` expose the resolved value. `App.availableLocales` is the union
of registered bundle locales and `appGlobals.availableLocales`.

## Translation Bundles

Bundles are flat key-to-message maps. They can be registered through:

```xml
<App
  locale="de-DE"
  localeBundles="{{
    'de-DE': {
      'hello': 'Hallo {name}',
      'inbox': '{count, plural, one {# Nachricht} other {# Nachrichten}}'
    }
  }}">
```

or at runtime:

```xml
<Button onClick="App.registerLocaleBundles('/locales/de-DE.json')">
  Load German
</Button>
```

Accepted bundle shapes:
- URL string or array of URL strings. Locale may be inferred from `*.locale.json`, or read from the JSON `locale` field.
- `{ locale, messages }`.
- `{ "de-DE": { key: message } }` locale map.

Missing keys emit `kind:"i18n"` / `code:"missing-key"` and return the key. Invalid ICU patterns emit
`code:"icu-parse-error"`. `strictI18n` escalates severity, but non-strict mode keeps fallback rendering.

## `<I18n>` Component

`<I18n key="...">` renders a bundle message. Extra props become variables; a leading `:` in the
prop name is stripped before formatting.

```xml
<I18n key="welcome" name="{loggedInUser.name}" />
<I18n key="inbox" count="{messages.length}" />
```

Named slots can replace placeholders such as `<link/>` in the translated message:

```xml
<I18n key="terms">
  <property name="link">
    <Link to="/terms">terms</Link>
  </property>
</I18n>
```

## Direction

`<App direction="auto">` uses `App.isRtlLocale(App.locale)` to set `document.documentElement.dir`.
Explicit `direction="ltr"` or `"rtl"` overrides auto detection.

## Built-In Strings

The first extracted framework strings are Form button defaults:

| Key | Default |
|---|---|
| `xmlui.form.cancel` | `Cancel` |
| `xmlui.form.save` | `Save` |
| `xmlui.form.saving` | `Saving...` |
| `xmlui.form.validating` | `Validating...` |

Custom Form label props still win. Only omitted defaults go through translation.

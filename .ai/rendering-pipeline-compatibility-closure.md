# Rendering Pipeline Compatibility Closure

Implemented slice:

- Added a rendering pipeline under `xmlui/src/runtime/rendering/`.
- Split runtime rendering into type definitions, binding evaluation,
  dependency normalization, reactive binding subscriptions, built-in renderers,
  user-defined component instantiation, and recursive node rendering.
- Reduced `xmlui/src/runtime/index.tsx` to public module creation, root
  store/scope initialization, render context creation, and root renderer
  invocation.
- Kept the compiler boundary intact: no rendering code is generated by the
  compiler.

Compatibility preserved now:

- `App` renders its children without an extra DOM wrapper.
- `H1` renders children in an `h1`.
- `Button` renders children when present, falls back to `label`, and attaches
  `onClick` through the generated event handler.
- User-defined component props are evaluated in the parent scope.
- User-defined component instances get isolated local state owners.
- Globals are shared through the runtime state store.
- Local variables shadow globals according to the runtime state model.
- Unknown components surface a clear render error.

Dependency and invalidation behavior:

- Compiler dependency metadata is normalized into runtime dependency keys.
- Local dependencies resolve to the owning runtime instance.
- Global dependencies resolve to app-global slots.
- `$props` dependencies are tracked as prop dependencies.
- React binding hooks subscribe to normalized state slots when dependency
  metadata is available and keep a broad revision fallback for metadata-free
  legacy descriptors.
- Lightweight binding evaluation counters are available for focused tests and
  future render-count profiling.

Intentional omissions:

- No slots, `when`, loaders, behavior chain, theming, layout context, component
  APIs, or provider stack.
- No custom DOM renderer or compiler-generated render functions.
- No full error boundary. Developer-facing render errors are minimal.
- Fine-grained recomputation is prepared through dependency subscriptions and
  instrumentation; full subtree scheduling remains a later optimization.

Next natural experiment:

- Minimal Managed React Contracts: define the tiny component metadata/contract
  surface for `App`, `H1`, `Button`, and user-defined components so compiler
  diagnostics can report unknown components, unknown events, and unsupported
  props earlier.

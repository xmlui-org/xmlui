# Component E2E Migration Requirement

Date: 2026-06-20

## Decision

A component migration is not complete until the corresponding original XMLUI
component E2E tests have been transferred or explicitly recorded as blocked
compatibility debt.

This means all old expanded test cases, including cases generated by loops in
the old spec files. Counting source-level `test(...)` calls is not sufficient.

## What Happened

During the Wave 4 primitive component slice, `Text` and `Heading`/`H1` were
initially migrated with new focused E2E tests, but the old component E2E suites
from `/Users/dotneteer/source/xmlui/xmlui/src/components` were not transferred.
That left important old behavior unverified.

The old tests uncovered real compatibility gaps:

- `Heading` and `H2`-`H6` had to be registered together with `H1`.
- Shortcut headings must accept a `level` prop and ignore it.
- `Text` and heading components must accept `id` so component APIs can be
  referenced from event handlers.
- Adapter API registration must be stable to avoid React update loops.
- `Heading maxLines > 1` needs multiline clamp behavior.
- `Text.hasOverflow()` must detect overflow against constrained parent layout.

## Rule For Future Waves

- From now on, every component migration must include migration of the existing
  original component E2E tests. This is part of the migration work itself, not a
  follow-up cleanup task.
- Inspect the original component folder before declaring a migrated component
  complete.
- Transfer all original component E2E tests into the migrated component folder,
  colocated with the implementation. If the old component has multiple spec
  files, migrate all of them.
- Run the migrated component E2E suite successfully before marking that
  component or wave complete. Existing old-suite skip markers may remain only
  when they are inherited deliberately and still documented by the old test.
- Keep the test intent and assertions as close to the original as the current
  harness allows.
- If a test cannot yet be transferred because another component or framework
  feature is missing, record the exact missing dependency and source test file
  in `.ai/` and keep it visible in the plan.
- New focused tests are useful, but they do not replace old-suite migration.
- Use the component E2E audit report to keep the count visible:
  `npm --workspace xmlui run compatibility:component-e2e-audit`.
  For active components, compare against the old Playwright expanded list. For
  example, Button has 159 old expanded tests across `Button.spec.ts` and
  `Button-style.spec.ts`.

## Current Transfer Pattern

Use the original spec filenames directly in the migrated component folder. Do
not create separate `*-old-e2e.spec.ts` files for copied original suites.

Examples:

- `xmlui/src/components/Text/Text.spec.ts`
- `xmlui/src/components/Heading/Heading.spec.ts`
- `xmlui/src/components/Heading/HeadingShortcuts.spec.ts`

## Correction: 2026-06-20

The earlier transferred old E2E suites were partial samples, not full old-suite
migrations. The first audit reported only 31 transferred old tests out of 1610
old component tests for currently mirrored component folders.

Going forward, a component status must remain incomplete until every old test is
ported and passing or each unported test is listed as blocked compatibility
debt with its missing dependency.

## Button Closure: 2026-06-20

The Button migration now uses the old spec filenames directly:

- `xmlui/src/components/Button/Button.spec.ts`
- `xmlui/src/components/Button/Button-style.spec.ts`

The temporary partial `Button-old-e2e.spec.ts` was removed. The Button audit now
reports 159 old tests, 159 transferred old tests, and 0 missing when run with
`--expanded=Button`. The migrated Button E2E suite passes with 153 passed and 6
skipped from the old suite's inherited skip markers.

## Status Update: 2026-06-20

The following copied original component suites are passing: Button, Text,
Heading/H1-H6, Card, Br, Fragment, and HtmlTags. The App original suite has
been copied but is not passing; it found 220 tests with only 3 passing in the
current implementation. Treat App as a separate incomplete migration wave.
